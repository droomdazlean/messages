Ultimate Responsive MessageBox System

By Droom Dazlean Solutions
This system lets you show beautiful, responsive message boxes on any web page using simple JavaScript functions.
No external libraries or HTML markup required.

How to Add the MessageBox System

  1. Click the Copy Script button below to copy the code to your clipboard.
  2. Paste it in your HTML page immediately after the <body> tag.
    For ASP.NET Web Forms: Paste it just after <form runat="server">.
    For standard HTML: Paste it after <body>.
  3. Call the message box functions anywhere in your page's scripts as shown below.

Available Functions

  • showMessage(title, message, icon, okButtonVisible, okButtonText, okButtonClickEvent)
  • showInfo(title, message, okButtonVisible, okButtonText, okButtonClickEvent)
  • showSuccess(title, message, okButtonVisible, okButtonText, okButtonClickEvent)
  • showError(title, message, okButtonVisible, okButtonText, okButtonClickEvent)

All functions are available globally after you add the script.

Parameter Reference

Parameter Type Description Optional?
title string Title at the top of the message box. If omitted, a default title is used (like "Information", "Success", or "Error"). Yes
message string/HTML Main content. Can be plain text or HTML (inputs, tables, etc.). No
icon SVG/HTML (showMessage only) Custom icon (SVG or HTML). Default icon is used if omitted. Yes
okButtonVisible boolean or string ("true"/"false") Show the OK button? Default: true for showMessage, false for others. Yes
okButtonText string Text for the OK button. Default: "OK". Yes
okButtonClickEvent string (JavaScript) JavaScript code to run when the OK button is clicked. Example: "alert('Clicked!')" Yes

How to Call Each Function (With Examples)

showMessage
showMessage("Welcome!", "This is a custom message box.");
showMessage("Notice", "You can add HTML here.", null, "true", "Continue", "alert('You clicked OK!')");
showInfo
showInfo("Just letting you know.");
showInfo("Info Title", "Info details here.", "true", "Close", "console.log('Info closed')");
showSuccess
showSuccess("Success!", "Your action was successful.");
showSuccess("Saved!", "Data saved.", "true", "Done", "alert('Saved!')");
showError
showError("Oops!", "Something went wrong.");
showError("Error", "Try again.", "true", "Retry", "location.reload()");

Features & Behavior

  • Responsive: Works on all devices and screen sizes.
  • Stackable: Open multiple messages in sequence; "Go Back" button appears for previous messages.
  • Dynamic Content: Put any HTML in the message, including forms and tables.
  • Custom Events: Run any JavaScript code when the OK button is clicked (as a string).
  • Closes easily: By OK button, close (X), or pressing ESC.
  • Preserves content: User input is saved if you go back and forth between messages.

Special Notes

  • If you only provide one string to showInfo, showSuccess, or showError, it is used as the message and a default title is applied.
  • For showMessage, you can also omit the icon and it will use a default one.
  • The OK button is hidden by default for showInfo, showSuccess, and showError. Set okButtonVisible to "true" to show it.
  • To execute multiple JS statements on OK, separate them with semicolons in the string.

Troubleshooting

  • Go Back button not showing? It only appears if you open a new message while another is open.
  • Custom JS not running? Make sure you pass your code as a string: "yourFunction();"
  • Message box not closing? Click OK, the close (X) button, or press ESC.
Tip: Use showMessage for general info, or showInfo, showSuccess, showError for themed messages with icons.
